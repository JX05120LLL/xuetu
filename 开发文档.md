# 学途在线教育平台 - 开发文档

## 1. 项目概述

学途在线教育平台是一个基于微服务架构的在线教育系统，包含用户前台系统和后台管理系统，实现课程展示、购买、学习和管理等功能。本项目旨在提供完整的在线教育解决方案，同时展示微服务架构的实现能力。

### 1.1 系统功能

- 用户认证与授权
- 课程分类与搜索
- 课程详情展示
- 订单管理与支付
- 学习进度记录
- 后台管理功能

## 2. 技术栈

### 2.1 后端技术

- **基础环境**：Java 17
- **核心框架**：Spring Boot 2.7.17、Spring Cloud 2021.0.8
- **微服务组件**：Spring Cloud Gateway、Nacos、OpenFeign、Sentinel
- **持久层技术**：MyBatis-Plus 3.5.3.2、MySQL 8.0

### 2.2 前端技术（可选实现）

- Element Plus
- Axios
- Vue Router

## 3. 系统架构

### 3.1 整体架构

学途平台采用微服务架构，通过网关统一入口，各个服务独立部署，通过服务注册与发现实现互相调用。

```
                    ┌─── 用户前台/后台系统 ───┐
                            │
                            ▼
                    ┌─── API网关(Gateway) ───┐
                            │
          ┌────────────────┬─────────────────┐
          │                │                 │
          ▼                ▼                 ▼
┌─── 用户服务 ───┐  ┌─── 课程服务 ───┐  ┌─── 订单服务 ───┐
      │                 │                  │
      └─────────────────┴──────────────────┘
                      │
                      ▼
               ┌─── 数据库 ───┐
```

### 3.2 服务划分

- **网关服务(gateway)**：请求路由、认证鉴权、限流
- **用户服务(user-service)**：用户管理、认证授权
- **课程服务(course-service)**：课程管理、分类、搜索
- **订单服务(order-service)**：订单管理、支付处理
- **公共服务(common-service)**：公共组件、工具类

#### 6.1.2 统一返回结果

```java
@Data
public class R<T> implements Serializable {
    private Integer code;
    private String message;
    private T data;
    
    public static <T> R<T> ok() {
        return ok(null);
    }
    
    public static <T> R<T> ok(T data) {
        R<T> r = new R<>();
        r.setCode(ResultCode.SUCCESS.getCode());
        r.setMessage(ResultCode.SUCCESS.getMessage());
        r.setData(data);
        return r;
    }
    
    public static <T> R<T> error(Integer code, String message) {
        R<T> r = new R<>();
        r.setCode(code);
        r.setMessage(message);
        return r;
    }
}
```

## 7. 数据库设计

数据库采用MySQL 8.0，包含以下主要表结构：

### 7.1 用户相关表

- `user`: 用户信息表
- `role`: 角色表
- `permission`: 权限表
- `user_role`: 用户角色关联表
- `role_permission`: 角色权限关联表

- `course`: 课程信息表
- `chapter`: 章节表

### 7.3 订单相关表

- `order`: 订单表
- `order_item`: 订单项表

## 10. 安全规范

1. 敏感信息加密存储（如用户密码使用BCrypt加密）
2. 接口权限控制，未授权接口返回401状态码
3. 防止SQL注入，使用参数绑定
4. XSS防御，对输入输出进行过滤
5. 敏感接口添加验证码或图形验证

## 11. 开发规范

### 11.1 命名规范

- 类名：使用大驼峰命名法，如`UserService`
- 方法名：使用小驼峰命名法，如`getUserById`
- 变量名：使用小驼峰命名法，如`userId`
- 常量名：全大写，单词间用下划线分隔，如`MAX_COUNT`
- 包名：全小写，如`com.star.user.controller`

### 11.2 注释规范

- 类注释：包含类的描述、作者、创建日期等
- 方法注释：包含方法的功能描述、参数说明、返回值说明等
- 关键代码注释：对于复杂的业务逻辑进行必要的注释说明

### 11.3 代码提交规范

- 提交说明格式：`[模块]：具体修改内容`
- 每次提交应当保证代码可运行，不要提交未完成的代码
- 提交前进行代码自检，确保没有明显的bug
