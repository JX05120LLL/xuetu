# 🚀 智慧教育平台测试启动指南

## 📊 服务端口配置总览

| 服务名称 | 端口 | 独立文档地址 | 启动命令 |
|---------|------|-------------|----------|
| 网关服务 | 8080 | - | `cd gateway && mvn spring-boot:run` |
| 用户服务 | 8088 | http://localhost:8088/doc.html | `cd user-service && mvn spring-boot:run` |
| 课程服务 | 8077 | http://localhost:8077/doc.html | `cd course-service && mvn spring-boot:run` |
| 订单服务 | 8055 | http://localhost:8055/doc.html | `cd order-service && mvn spring-boot:run` |
| 学习服务 | 8044 | http://localhost:8044/doc.html | `cd learning-service && mvn spring-boot:run` |

## 🎯 两种测试方式

### 方式1: 各服务独立测试 (开发推荐)

#### 1.1 启动单个服务测试
```bash
# 测试用户服务
cd user-service
mvn spring-boot:run

# 访问用户服务文档
http://localhost:8088/doc.html

# 直接测试用户接口
POST http://localhost:8088/auth/login
{
  "username": "admin",
  "password": "123456"
}
```

#### 1.2 启动学习服务测试
```bash
# 测试学习服务
cd learning-service
mvn spring-boot:run

# 访问学习服务文档
http://localhost:8084/doc.html

# 直接测试学习接口
POST http://localhost:8084/learning/record/progress
{
  "courseId": 1,
  "lessonId": 1,
  "progress": 300
}
```

### 方式2: 网关统一测试 (生产环境)

#### 2.1 启动所有服务
```bash
# 按顺序启动 (建议开5个终端)
cd user-service && mvn spring-boot:run
cd course-service && mvn spring-boot:run  
cd order-service && mvn spring-boot:run
cd learning-service && mvn spring-boot:run
cd gateway && mvn spring-boot:run
```

#### 2.2 访问统一文档
```bash
# 网关聚合文档
http://localhost:8080/doc.html

# 文档中心首页
http://localhost:8080/api-docs
```

#### 2.3 通过网关测试接口
```bash
# 用户登录 (通过网关)
POST http://localhost:8080/api/user/auth/login
{
  "username": "admin", 
  "password": "123456"
}

# 学习进度更新 (通过网关)
POST http://localhost:8080/api/learning/record/progress
{
  "courseId": 1,
  "lessonId": 1,
  "progress": 300
}
```

## 🔍 故障排查步骤

### 步骤1: 检查单个服务启动
```bash
# 1. 检查用户服务
cd user-service && mvn spring-boot:run
# 看到 "用户服务启动成功" 后访问:
curl http://localhost:8088/doc.html

# 2. 检查学习服务  
cd learning-service && mvn spring-boot:run
# 看到 "学习服务启动成功" 后访问:
curl http://localhost:8084/doc.html
```

### 步骤2: 检查网关路由
```bash
# 启动网关
cd gateway && mvn spring-boot:run

# 检查路由是否生效
curl http://localhost:8080/api/user/v3/api-docs
curl http://localhost:8080/api/learning/v3/api-docs
```

### 步骤3: 检查Nacos注册
```bash
# 访问Nacos控制台
http://localhost:8848/nacos

# 检查服务列表是否包含:
- user-service
- course-service  
- order-service
- learning-service
```

## 🚨 常见问题解决

### 问题1: 端口被占用
```bash
# 查看端口占用
netstat -ano | findstr :8088
netstat -ano | findstr :8084

# 杀死进程
taskkill /PID <进程ID> /F
```

### 问题2: 无法访问doc.html
**检查项目**:
- ✅ 服务是否启动成功
- ✅ 端口是否正确
- ✅ Knife4j依赖是否正确

**解决方案**:
```bash
# 重新编译
mvn clean compile

# 重新启动
mvn spring-boot:run
```

### 问题3: 网关路由不通
**检查项目**:
- ✅ 所有服务是否注册到Nacos
- ✅ 网关路由配置是否正确
- ✅ 服务名称是否匹配

## 🎪 快速验证脚本

### Windows批处理
```bat
@echo off
echo 检查所有服务状态...
curl -s http://localhost:8088/actuator/health | findstr "UP"
curl -s http://localhost:8077/actuator/health | findstr "UP"  
curl -s http://localhost:8055/actuator/health | findstr "UP"
curl -s http://localhost:8084/actuator/health | findstr "UP"
curl -s http://localhost:8080/actuator/health | findstr "UP"
echo 检查完成！
```

### Linux/Mac脚本
```bash
#!/bin/bash
echo "检查所有服务状态..."
services=(8088 8077 8055 8084 8080)
for port in "${services[@]}"; do
    if curl -s http://localhost:$port/actuator/health | grep -q "UP"; then
        echo "✅ 端口 $port 服务正常"
    else
        echo "❌ 端口 $port 服务异常"
    fi
done
```

## 📈 推荐测试流程

### 第一次测试 (建议)
1. **单独启动用户服务** → 测试登录接口
2. **单独启动学习服务** → 测试学习接口  
3. **启动网关** → 测试路由转发
4. **完整集成测试** → 验证端到端流程

### 日常开发测试
1. **只启动在开发的服务** → 快速调试
2. **需要时启动依赖服务** → 测试接口调用
3. **定期全量测试** → 确保集成正常

---

**🎯 总结**: 你可以根据需要选择测试方式，开发时用独立测试，集成时用网关测试！
