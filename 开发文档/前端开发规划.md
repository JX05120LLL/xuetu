# 学途在线教育平台 - 前端开发规划文档

## 📋 项目概述

### 项目名称
学途在线教育平台前端系统

### 系统划分
1. **用户端系统** (xuetu-web) - 面向学生的在线学习平台
2. **管理端系统** (xuetu-admin) - 面向管理员的后台管理系统

### 技术栈
- **框架**: Vue 3 + TypeScript
- **构建工具**: Vite
- **UI组件库**: 
  - 用户端: Element Plus
  - 管理端: Element Plus
- **状态管理**: Pinia
- **路由**: Vue Router 4
- **HTTP请求**: Axios
- **CSS方案**: SCSS + 部分TailwindCSS
- **图标**: @element-plus/icons-vue

## 🔌 后端服务接口说明

### 服务端口配置（重要！）

| 服务名称 | 端口 | 访问路径前缀 | 说明 |
|---------|------|-------------|------|
| 网关服务 | 8080 | - | **推荐通过网关访问所有服务** |
| 用户服务 | 8088 | /user | 用户认证、用户管理 |
| 课程服务 | 8077 | /course | 课程、分类、章节、评论 |
| 订单服务 | 8033 | /api | 订单、支付、用户课程 |
| 学习服务 | 8044 | /api | 学习记录、笔记 |
| AI服务 | 8065 | /api | AI对话、推荐、分析 |
| 管理服务 | 8055 | /api | 角色权限管理 |

### 统一响应格式
```typescript
interface ApiResponse<T> {
  code: number;      // 200成功，其他失败
  message: string;   // 响应消息
  data: T;          // 响应数据
}
```

### 分页请求格式
```typescript
interface PageParam {
  current: number;   // 当前页码，从1开始
  size: number;      // 每页大小
}

interface PageResult<T> {
  records: T[];      // 数据列表
  total: number;     // 总记录数
  current: number;   // 当前页
  size: number;      // 每页大小
}
```

### 3.2 Axios封装（request.ts）

```typescript
import axios from 'axios'
import { ElMessage } from 'element-plus'
import { useUserStore } from '@/stores/user'

const request = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL,
  timeout: 10000
})

// 请求拦截器
request.interceptors.request.use(
  config => {
    const userStore = useUserStore()
    if (userStore.token) {
      config.headers.Authorization = `Bearer ${userStore.token}`
    }
    return config
  },
  error => {
    return Promise.reject(error)
  }
)

// 响应拦截器
request.interceptors.response.use(
  response => {
    const res = response.data
    if (res.code !== 200) {
      ElMessage.error(res.message || '请求失败')
      // 401 未授权，跳转登录
      if (res.code === 401) {
        const userStore = useUserStore()
        userStore.logout()
      }
      return Promise.reject(new Error(res.message || '请求失败'))
    }
    return res.data
  },
  error => {
    ElMessage.error(error.message || '网络错误')
    return Promise.reject(error)
  }
)

export default request
```

## 📝 五、开发规范

### 5.1 命名规范

- **文件命名**: PascalCase (如: UserProfile.vue)
- **组件命名**: PascalCase (如: CourseCard)
- **路由命名**: kebab-case (如: /user-center)
- **API函数**: camelCase (如: getUserInfo)
- **常量**: UPPER_SNAKE_CASE (如: API_BASE_URL)

### 5.2 代码规范

- 使用 ESLint + Prettier
- 使用 TypeScript 严格模式
- 组件必须有注释说明
- 复杂逻辑必须加注释
- API调用必须有错误处理

### 5.3 Git提交规范

```
feat: 新功能
fix: 修复bug
docs: 文档更新
style: 代码格式调整
refactor: 重构
test: 测试相关
chore: 构建/工具链相关
```

### 5.4 API调用规范

```typescript
// ✅ 正确示例
try {
  const data = await getCourseList({ current: 1, size: 10 })
  // 处理数据
} catch (error) {
  console.error('获取课程列表失败', error)
  ElMessage.error('获取课程列表失败')
}

// ❌ 错误示例 - 没有错误处理
const data = await getCourseList({ current: 1, size: 10 })
```

## ⚠️ 注意事项

### 1. 端口号注意
- **严格遵守后端端口配置**
- **推荐统一通过网关 (8080) 访问**
- 避免直连微服务端口

### 2. 请求格式注意
- **严格按照后端返回的数据结构编写类型定义**
- 分页参数: `current` 和 `size` (不是 page 和 pageSize)
- 时间字段: `createdTime` 和 `updatedTime` (驼峰命名)

### 3. 权限处理
- 所有需要登录的页面添加路由守卫
- Token失效自动跳转登录
- 管理端根据权限动态显示菜单

### 4. 错误处理
- 所有API调用必须有 try-catch
- 统一的错误提示
- 特殊错误码特殊处理 (如401)

### 5. 性能优化
- 路由懒加载
- 图片懒加载
- 组件按需引入
- 合理使用keep-alive
