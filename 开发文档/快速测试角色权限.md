```bash

cd D:\code\project\xuetu\admin-service

cd D:\code\project\xuetu\gateway
```

等待所有服务启动成功（看到"启动成功"日志）

## ✅ 核心功能测试（3分钟）

### 测试1: 管理员登录 ⭐⭐⭐⭐⭐

**请求**:
```bash
POST http://localhost:8080/api/user/auth/login
Content-Type: application/json

{
  "username": "admin",
  "password": "admin123"
}
```

**期望结果**: ✅ 返回包含roles和permissions
```json
{
  "code": 200,
  "data": {
    "token": "eyJ...",
    "userInfo": {
      "username": "admin",
      "roles": ["SUPER_ADMIN"]  // ✅ 有角色
    },
    "permissions": [              // ✅ 有权限
      "course:create",
      "order:view:all",
      ...
    ]
  }
}
```

**如果失败**: 检查admin-service是否启动，SQL是否执行成功

### 测试3: 权限验证 - 学生创建课程（应该失败）⭐⭐⭐⭐⭐

**步骤**:
1. 用已有的普通用户登录（或创建新用户）
2. 获取token
3. 尝试创建课程

**请求**:
```bash
# 假设已获取student的token
POST http://localhost:8080/api/course/create

{
  "title": "测试课程"
}
```

**期望结果**: ❌ 403 Forbidden
```json
{
  "code": 403,
  "message": "权限不足，无法访问该资源"
}
```

**✅ 如果返回403，说明权限验证工作正常！**

## 🐛 常见问题快速解决

### 问题1: 登录后roles为空
```bash
# 检查1: admin-service是否启动
curl http://localhost:8055/actuator/health

# 检查2: 数据库是否有数据
mysql> SELECT * FROM user_role WHERE user_id = 100;

# 解决: 重新执行SQL
mysql -u root -p xuetu_db < sql/rbac_init_data.sql
```

### 问题2: 权限验证不生效
```bash
# 检查Gateway日志
# 应该看到：
# "权限验证: /api/course/create"
# "用户角色: [STUDENT]"
# "需要的角色: [TEACHER, ADMIN, SUPER_ADMIN]"
# "权限不足"

# 如果没有日志，检查AuthorizationFilter是否加载
```

### 问题3: Token无效
```bash
# 检查JwtUtil是否支持Map claims
# 确保有这个方法：
# JwtUtil.generateToken(Map<String, Object> claims, long expireTime)
```
