## 🚀 需要重启网关服务

### 方式一：使用本地模式（推荐）

**在IDE中重启网关服务：**

1. **停止当前的GatewayApplication**
2. **修改启动配置**：
   - 找到 `GatewayApplication` 的Run Configuration
   - 添加环境变量或VM参数：
     ```
     -Dspring.profiles.active=local
     ```
   - 或者在 Program arguments 中添加：
     ```
     --spring.profiles.active=local
     ```
3. **启动服务**

### 方式二：使用默认模式（需要Nacos）

如果你已经启动了Nacos（8848端口），可以直接重启网关：

1. **停止当前的GatewayApplication**
2. **直接启动GatewayApplication**

## 🔍 故障排查

### 问题1：网关启动失败

**错误信息**：`Connection refused: localhost/127.0.0.1:8848`

**原因**：默认配置依赖Nacos，但Nacos未启动

**解决方案**：
```bash
# 使用本地模式启动
--spring.profiles.active=local
```

### 问题2：仍然404

**检查清单**：
- [ ] 网关服务已重启？
- [ ] 后端服务（8077, 8088, 8033）已启动？
- [ ] 前端项目已重启？（`npm run dev`）
- [ ] 浏览器缓存已清除？

### 问题3：前端请求超时

**原因**：后端服务未启动或响应慢

**解决方案**：
1. 检查后端服务日志
2. 确认数据库已启动（MySQL）
3. 检查Redis是否启动（如果使用）
